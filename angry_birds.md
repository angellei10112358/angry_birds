## 0. 黄金原则（最重要，先读这一节）

1. **先算数，再写码。** 任何涉及距离、速度、重力、力度的参数，必须先用第 4、9
   节的公式手动计算出取值范围，确认"最大力度一定能打到最远的敌人，且留有余量"，
   然后才能把数值写进代码。
2. **所有魔法数字必须可追溯。** 每一个物理常量（重力、最大速度、弹弓拉伸上限、
   材质血量等）都要在代码注释里写明：它的单位是什么、为什么取这个值、由哪个
   约束推导而来。
3. **不允许"拍脑袋"取值。** 例如不允许写 `maxSpeed = 30` 而不解释 30 怎么来的。
4. **交付即自检。** 交付时必须附上第 9 节清单的计算结果。

## 1. 项目目标与托管约束

- 目标：实现一个可玩的 Angry Birds 类弹弓物理游戏。
- 托管平台：**GitHub Pages（纯静态）**。因此：
  - 必须是纯前端，无后端、无构建步骤即可运行（或提供已构建产物）。
  - 入口为 `index.html`，可直接双击或通过静态服务器打开即运行。
  - 不依赖任何需要服务器的 API。
- **存储限制：** 不依赖 `localStorage`/`sessionStorage` 作为唯一数据源（部分
  环境不可靠）。关卡进度优先用内存变量；如要持久化，做成"可降级"——存储失败
  时不报错、不崩溃。

## 2. 技术栈

- 语言：HTML + CSS + 原生 JavaScript（ES Modules）。
- 物理引擎：**Matter.js**（通过 CDN 或本地 vendored 文件引入）。
  - 推荐 CDN：`https://cdn.jsdelivr.net/npm/matter-js/build/matter.min.js`
  - 为保证 GitHub Pages 离线可用，建议同时把该文件 vendored 到 `/vendor/`。
- 渲染：使用 `<canvas>`。可用 Matter.js 自带 Renderer 起步，但最终建议自定义
  渲染以便绘制小鸟、积木贴图与瞄准线。
- 禁止引入其他重型框架。

## 3. 坐标系与单位约定（必须统一）

为避免"单位混乱导致打不到敌人"，全项目统一如下约定，**不得在不同模块使用不同单位**：

- 世界尺寸：`WORLD_W = 1280`，`WORLD_H = 720`（像素，px）。canvas 用 CSS 等比缩放
  适配屏幕，但**物理世界坐标恒定为 1280×720 px**，所有计算都在世界坐标下进行。
- 时间步长：固定 60 FPS，单步 `dt = 1/60 s ≈ 0.0167 s`。使用固定步长更新物理
  （fixed timestep），避免不同设备帧率影响手感与数值。
- 速度单位：**px/step**（Matter.js 的 `Body.setVelocity` 即为每步位移像素数）。
  本文所有速度阈值均以 px/step 表达，换算关系：`px/s = (px/step) × 60`。
- 重力单位：Matter 的 `engine.gravity.y` 是"缩放后加速度"。**约定** `engine.gravity.scale`
  保持默认，使有效重力加速度 `g`（单位 px/step²）= `gravity.y_value`。
  我们在第 4 节直接以 `g`（px/step²）参与抛体公式。
- 角度：弧度制，0 朝右（+x），逆时针为正。

## 4. 弹弓与抛体物理（核心，附验算公式）

### 4.1 弹弓机制

- 弹弓锚点（小鸟起飞点）：`ANCHOR = { x: 220, y: 480 }`（在世界左侧地面附近）。
- 玩家从锚点向后下方拖拽，松手后小鸟以拖拽的反方向被弹出。
- 拖拽位移向量 `d = pointer - ANCHOR`，发射方向为 `-d` 的单位向量。
- 拖拽距离 `L = |d|`，并**钳制**到最大拉伸 `L_max`（见下）。
- 发射初速度大小：`v0 = (L / L_max) × V_MAX`，方向 `-d̂`。
  - 即拉满（L = L_max）时速度恰为 `V_MAX`。

### 4.2 关键参数（agent 必须按约束推导，不得照抄示例）

| 参数 | 含义 | 单位 | 约束 |
|---|---|---|---|
| `V_MAX` | 拉满时初速度大小 | px/step | 见 4.3 必须满足射程约束 |
| `L_max` | 弹弓最大拉伸距离 | px | 建议 120–180，影响手感不影响射程 |
| `g` | 有效重力加速度 | px/step² | 建议 0.8–1.5，先定 g 再解 V_MAX |
| `SPEED_CAP` | 任意刚体允许的最大速度 | px/step | 防隧穿，见第 6 节 |

### 4.3 射程公式与"必须打得到"硬约束（核心验算）

理想抛体（忽略空气阻力）的水平射程：

    R(θ) = v0² · sin(2θ) / g          （单位：px，v0 单位 px/step，g 单位 px/step²）

最大射程在 θ = 45°：

    R_max = V_MAX² / g

**硬约束 A（必须打得到最远敌人）：**

设最远敌人与锚点的水平距离为 `D_far = x_enemy_far - ANCHOR.x`。
要求拉满且以最优角度发射时，射程必须明显覆盖最远敌人并留有余量：

    R_max ≥ SAFETY × D_far ，  取 SAFETY = 1.3

即：

    V_MAX ≥ sqrt( 1.3 × D_far × g )

> Agent 操作步骤：
> 1. 先确定敌人放置区的最大 x（见第 5 节关卡定义），算出 D_far。
> 2. 选定 g（建议先取 1.0）。
> 3. 用上式解出 V_MAX 的下界，向上取整留余量后写入代码。
> 4. 回代验证 R_max = V_MAX²/g ≥ 1.3 × D_far，并在自检表中给出数值。

**硬约束 B（高度可达）：**
若敌人/结构有高度 `H_target`，最大可达高度（θ=90°）为 `H_peak = V_MAX²/(2g)`。
要求 `H_peak ≥ 1.3 × H_target`。通常 A 满足时 B 自动满足，但仍须在自检中验算。

**硬约束 C（不要快到隧穿）：**
单步位移不得超过最薄碰撞体厚度的一半（见第 6 节）。即
`V_MAX ≤ SPEED_CAP` 且 `SPEED_CAP < min_collider_thickness / 2`。
若 A 要求的 V_MAX 与 C 冲突，**解决顺序**为：先减小 g（增大射程效率），
再增大碰撞体厚度，最后才考虑开启 CCD（连续碰撞检测）/子步进。

## 5. 关卡与场景定义

- 地面：底部静态刚体，`y ≈ 660`，覆盖整个宽度。
- 弹弓在左侧（见 4.1）。
- 敌人（猪）放置区：建议 `x ∈ [820, 1180]`，`y` 在地面之上由结构决定。
  - 因此默认 `D_far = 1180 - 220 = 960 px`（agent 必须用实际关卡数据复算）。
- 关卡用数据结构描述，便于扩展，例如：

```js
const LEVELS = [
  {
    birds: ['red', 'yellow', 'blue'],     // 本关可用小鸟队列（顺序）
    blocks: [ /* {x,y,w,h,material,angle} */ ],
    pigs:   [ /* {x,y,r} */ ],
  },
  // ...
];

至少实现 3 个关卡，难度递增。
6. 碰撞稳定性（防穿模）
设最薄碰撞体厚度为 MIN_THICK（如玻璃板厚度）。约定 MIN_THICK ≥ 16 px。
SPEED_CAP < MIN_THICK / 2。每步更新后对所有刚体速度做钳制，超出则按比例缩到
SPEED_CAP。
若硬约束 A 要求的 V_MAX 接近或超过 SPEED_CAP：
优先降低 g（射程 ∝ 1/g），在 g ∈ [0.8,1.5] 内调整；
仍不够则加厚碰撞体（提高 MIN_THICK）；
最后启用子步进：每渲染帧把物理更新拆成 N 个子步，每子步位移 = 速度/N，
从而单子步位移 < MIN_THICK/2。在自检中记录所选方案与最终单步最大位移。
启用 Matter 的 sleeping（enableSleeping: true），静止物体休眠以省性能。
7. 必须实现的核心功能
拖拽发射 + 力度/方向 —— 见第 4 节，含拉伸钳制。
瞄准辅助线 —— 拖拽时用抛体公式逐点采样画虚线轨迹（用与物理一致的 g、v0、
dt 迭代），让预测线与实际飞行轨迹吻合（误差应很小，见自检）。
多种小鸟（至少 3 种，点击屏幕触发能力）：
red 普通；
yellow 点击后沿当前速度方向加速冲刺（速度乘以系数，但钳制到 SPEED_CAP）；
blue 点击后分裂成 3 只（速度方向小角度扇形散开）。
可选 bomb（点击/触地爆炸，对范围内刚体施加冲量）。
材质系统：木/玻璃/石头，各有不同血量与受击减血。受击伤害可基于碰撞冲量
（pair 的相对速度 × 质量）计算。玻璃血量低、石头高。
猪的血量与消灭判定：受冲量超过阈值掉血，归零则消灭。
关卡结束判定（易出 bug，按下列逻辑实现）：
胜利：所有猪被消灭。
失败：小鸟队列用尽 且 仍有猪存活。
必须加稳定延迟：触发判定前等待场景"基本静止"（所有刚体速度 < 阈值，
或固定 1.5–2.0 秒），避免物体还在滚动时误判。
星级评分：按剩余小鸟数 / 得分给 1–3 星。
关卡选择 + 进度（内存优先，存储可降级）。
音效与粒子（撞击碎片、爆炸火花、过关动画）——可用 WebAudio 合成音效，
不依赖外部音频 URL。
